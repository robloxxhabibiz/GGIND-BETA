Hubungkan ke Supabase — gratis, aman, dan data dilindungi RLS

Berbeda dari Upstash, Supabase anon key aman dipublikasikan. Row Level Security (RLS) di server yang memastikan setiap user hanya bisa mengubah data miliknya sendiri.
1
Buat akun gratis di supabase.com → buat project baru → tunggu hingga selesai (~2 menit)
2
Di Supabase dashboard, klik SQL Editor → klik New query → paste SQL berikut → klik Run
-- ① Tabel profil user CREATE TABLE public.profiles ( id UUID REFERENCES auth.users ON DELETE CASCADE PRIMARY KEY, username TEXT UNIQUE NOT NULL, display_name TEXT NOT NULL, bio TEXT DEFAULT '', website TEXT DEFAULT '', location TEXT DEFAULT '', created_at TIMESTAMPTZ DEFAULT NOW() ); -- ② Tabel repository CREATE TABLE public.repositories ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, owner_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL, owner_username TEXT NOT NULL, name TEXT NOT NULL, description TEXT DEFAULT '', is_public BOOLEAN DEFAULT TRUE, topics TEXT[] DEFAULT '{}', stars_count INTEGER DEFAULT 0, forks_count INTEGER DEFAULT 0, forked_from TEXT DEFAULT NULL, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(owner_id, name) ); -- ③ Tabel file CREATE TABLE public.files ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, repo_id UUID REFERENCES public.repositories(id) ON DELETE CASCADE NOT NULL, name TEXT NOT NULL, content TEXT DEFAULT '', updated_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(repo_id, name) ); -- ④ Tabel stars CREATE TABLE public.stars ( user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE, repo_id UUID REFERENCES public.repositories(id) ON DELETE CASCADE, PRIMARY KEY (user_id, repo_id) ); -- ⑤ Aktifkan RLS (Row Level Security) ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY; ALTER TABLE public.repositories ENABLE ROW LEVEL SECURITY; ALTER TABLE public.files ENABLE ROW LEVEL SECURITY; ALTER TABLE public.stars ENABLE ROW LEVEL SECURITY; -- ⑥ Kebijakan akses Profiles CREATE POLICY "profiles_select" ON public.profiles FOR SELECT USING (true); CREATE POLICY "profiles_insert" ON public.profiles FOR INSERT WITH CHECK (auth.uid() = id); CREATE POLICY "profiles_update" ON public.profiles FOR UPDATE USING (auth.uid() = id); -- ⑦ Kebijakan akses Repositories CREATE POLICY "repos_select" ON public.repositories FOR SELECT USING (is_public = true OR owner_id = auth.uid()); CREATE POLICY "repos_insert" ON public.repositories FOR INSERT WITH CHECK (owner_id = auth.uid()); CREATE POLICY "repos_update" ON public.repositories FOR UPDATE USING (owner_id = auth.uid()); CREATE POLICY "repos_delete" ON public.repositories FOR DELETE USING (owner_id = auth.uid()); -- ⑧ Kebijakan akses Files CREATE POLICY "files_select" ON public.files FOR SELECT USING ( EXISTS (SELECT 1 FROM public.repositories r WHERE r.id = repo_id AND (r.is_public OR r.owner_id = auth.uid())) ); CREATE POLICY "files_all" ON public.files FOR ALL USING ( EXISTS (SELECT 1 FROM public.repositories r WHERE r.id = repo_id AND r.owner_id = auth.uid()) ); -- ⑨ Kebijakan akses Stars CREATE POLICY "stars_select" ON public.stars FOR SELECT USING (true); CREATE POLICY "stars_insert" ON public.stars FOR INSERT WITH CHECK (auth.uid() = user_id); CREATE POLICY "stars_delete" ON public.stars FOR DELETE USING (auth.uid() = user_id);
3
Buka Project Settings → API → copy Project URL dan anon public key
4
Di Authentication → Settings, nonaktifkan "Confirm email" agar user bisa langsung login tanpa verifikasi email
